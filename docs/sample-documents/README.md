# Vendor Onboarding SOP Sample Documents

These files are fictional learning fixtures for ReviewFlow AI. They do not describe the policy of a real organization and must not be used as legal, financial, security, procurement, or compliance guidance.

## Files

| File | Purpose |
| --- | --- |
| `vendor-onboarding-sop-v1.md` | The fictional current approved SOP (baseline). |
| `vendor-onboarding-sop-v2.md` | A fictional proposed revision containing intentional changes. |
| `expected-material-changes.md` | The ground-truth comparison manifest for future tests and AI evaluation. |

## How to use these fixtures

1. Treat V1 as the approved baseline.
2. Treat V2 as an unapproved proposed revision.
3. Compare V1 and V2 without using the ground-truth file.
4. Compare the detected changes with `expected-material-changes.md` afterward.

The expected-change manifest distinguishes substantive changes from non-material editorial changes. It is intentionally human-authored so future AI output can be evaluated against a known reference.

## Scope note

ReviewFlow reviews changes to the SOP itself. It does not onboard vendors, collect vendor banking or tax information, perform sanctions screening, or make automated approval decisions.
